General Questions
1.   I am reviewing some code that spawns a thread; specifically the pthread_create() API. When looking at this API, what is one of the required parameters I should see used?
  Answer:   The two required parameters are a reference to the thread ID and a function pointer to the thread "main()" function. Thread options and function arguments are both optional items.
  Notes:   This is a question to start with that is designed to set the pace for questions that follow. It is not difficult at all if you know the API, and really not difficult if you think about the API. This API is documented here.
  Applicability:   This is about as simple as the API to threads gets. It is an attempt to see if the candidate has worked in the API at all and to start the interview with a set direction in questions. Some of the harder questions should sit up front to intentionally frustrate the candidate and punch a hole in their confidence. (We are looking for people that press on through this part of the questions.) While this question is not in itself by any means "devastating" in this respect, it is designed to break from an "admin" interview.
 
2.   I am designing a replacement for an I/O intensive system. It does some sequential I/O to one set of disks, random I/O to another set, and frequently moves data back and forth between some of the differing sets of disks on the system. The current system uses HDDs. I am considering SSDs for the new system, what are some considerations of this change that I should be aware of?
  Answer:   Some key differences that may be discussed:
  • SSDs are "faster" and more expensive are obvious answers.
  • SSDs will not scale the same as HDDs, RAID stripes will behave differently.
  • Write amplification will be a problem (particularly on small writes).
  • Not all SSD technology is equal in price or performance.
  • Performance may degrade over time on busy write disks.
  Notes:   I am looking for someone who does not view SSDs as a black box / object.
  Applicability:   This is an indication that the candidate has worked with, or thought about the technology. Additionally, this question lays down some factoids that the candidate needs to sort through and not get hung up on.
 
3.   I have a process that is using shared memory. For what reason would an application use shared memory? --follow up--> What are some other IPC mechanisms that the process may be using?
  Answer:   This is a means of interprocess communication. This is (primarily) so two applications can access the same piece of memory. Because the type and method of shared memory is not specified, it can have additional uses - one popular method is to mmap() a file to make it look like a location in memory, rather than using read() and write() calls.
  Notes:   This is a trial question. I am curious as to what some applicants will say. I would expect that someone who describes the purpose of shared memory has investigated why it exists - and this may be indicative of curiosity about the technology.
  Applicability:   This is a product / technology question. Product people may answer Oracle uses it, technology people will discuss the actual purpose. (This is a question left over (unchanged) from the admin interview.)
 
4.   I see two syscalls being called heavily on a system I am monitoring. One is fork() and the other is exec(). What is the primary difference between these two commands? --follow up--> What is happening on this system?
  Answer:   fork() creates a new process, exec() modifies the purpose and nature of the existing process.
  Notes:   This may prompt a discussion on why, or the point of the high rates of the two syscalls being called. This offers an opportunity to let the candidate talk a bit about what this means.
  Applicability:   Strictly about weeding out someone who does not understand the concepts. This is a fundamentally basic question that all should get right. In the admin interviews I found (even talented) candidates did not know this simple question. This is William's version of "What is file magic?".
 
5.   I have an application that manages a large amount of data and is extremely performance sensitive. I would like to put the entire data set into memory, but that is not possible due to the size of the data. What can I do to solve this problem?
  Answer:   This is a very open question that has many possible answers. Some possible answers / discussion directions:
  • Map Reduce or similar big data parallelization
  • SSDs as a tiered memory solution
  • Indexing or organizing data for faster retrieval
  Notes:   On questions like this I like to let the candidate take the line in any direction they want (or at least not in a specific direction). For example if the candidate asks about specifics about X, I reply with a question that asks for the implications of X - having the candidate first relate that he knows the forks in the road, not just one path to the solution.
  Applicability:   This is a thought problem that indicates the candidates ability to brainstorm complex issues.
 
6.   I am working in a new language that has a dynamic array data type. I am not required to specify the size of the array when creating it. What might be a good way of implementing such a type so that either performance or simplicity is achieved?
  Answer:   Multiple answers exist - some discussion points:
  • linked list - expensive in offset lookups, simple to implement
  • simple array - simple to add items, predictable lookup, occasional expensive copy-on-add operations
  • linked list of arrays - balanced approach but dependent on "chunk" size
  Notes:   Free form opportunity to discuss data structures.
  Applicability:   Shows knowledge of language internals.
 
7.   I just ran traceroute to a remote host. How does it find the intermediate routers, and how does it get them to respond?
  Answer:   traceroute sends multiple packets with differing TTL values. The packets time (hop-count) out at different parts of the network.
  Notes:   I tend to dwell on this question, often re-asking in multiple ways... "How does this binary find its way through the network, does it have some sort of database it queries?", "So if this is in the routing table, then the routing table must include entries for the third, fourth, and fifth hops?" The question is about understanding networking and how these tools work. The point is that traceroute does not know (beforehand) some map of the network. It is also important to understand that traceroute does not necessarily generate an accurate map of the path to the remote host.
  Applicability:   A proper understanding of networking can be challenging for admins to grasp - most will not properly answer this. (This is a question left over (unchanged) from the admin interview.)
 
8.   I want to open a file. I see that I can use fopen() and open(). What are some differences between these two calls?
  Answer:   fopen() is library based, has the concept of buffering, and has "higher level" read and write calls associated with it. open() is a system call (possibly thinly wrapped by library) that returns a different type of integer file descriptor than a file handle pointer, is designed with read() and write() in mind - both lower level calls.
  Notes:   This is an attempt to have a question 8.
  Applicability:   This should be quite fundamental for qualified candidates.
 
9.   I have created a variable that is on the stack, and another that is on the heap. How did I do this? Follow-up --> What are the implications of each?
  Answer:   Not looking for anything complex here such as globals or BSS... Just looking to hear that one was allocated from memory (heap) and the other was declared in scope. One can be seen from anywhere (including other threads), the other is transient and scoped.
  Notes:   Another basic filtering question.
  Applicability:   Basic understanding of process memory.
 
10.   I want to put regular and frequent wall-clock timestamps in my application output. I need to know what the system must do in order to generate these values. How is this done?
  Answer:   Candidate should relate:
  1. Get Unix epoch offset (Unix system time) via the time() call.
  2. Convert this time into a localized offset.
  3. Subtract any leap seconds (if looking to past).
  4. Convert the offset to localized representations (strings)
Also:
  • Time server is not (directly) part of this process.
  • Posix and Olsen are different here. TZ or OS specific implementations.
  • gettimeofday() and clock_gettime() are higher resolution options.
  Notes:   Looking here for the candidate to walk through the process.
  Applicability:   Candidates should know how time is handled in the system.
 
Linux Specific Questions
11.   I want to define (or instantiate) an integer in bash, how can I do that?
  Answer:   Bash uses the "declare" keyword with the -i option for integers. Bash also accepts the Korn "typeset" with the same -i option. Additionally, Korn's typeset -i offers an option for a base modifier.
  Notes:   Either key word is correct. The answer may indicate a background in Korn or more pure Linux.
  Applicability:   Whatever the answer, there is an expectation that a shell developer declares his variables (or at least know how).
 
12.   I have these two tunables on my system; rmem_max and rmem_default. What is the difference between these two and, how do I check them?
  Answer:   Those parameters set socket rx buffer size and maximum size, if socket is open without setsockopt() (SO_RCVBUF) the kernel assigns socket memory double of rmem_default up to double of rmem_max, if it app uses SO_RCVBUF option the kernel assigns double of requested up to double of rmem_max.
  Notes:   Set (via sysctl) with net.core.rmem_max and net.core.rmem_default.
  Applicability:   Understanding of Linux socket tunables.
 
13.   I have heard discussion of a tickless kernel. What does that mean? What is the alternative to a tickless kernel, or even is there an alternative?
  Answer:   Tickless kernel, dynamic ticks or NO_HZ is a config option that enables a kernel to run without a regular timer tick. The timer tick is a timer interrupt that is usually generated HZ times per second, with the value of HZ being set at compile time and varying between around 100 to 1500. Running without a timer tick means the kernel does less work when idle and can potentially save power because it does not have to wake up regularly just to service the timer. The configuration option is CONFIG_NO_HZ and is set by Tickless System (Dynamic Ticks), on the Kernel Features configuration menu.
  Notes:   CONFIG_NO_HZ was replaced in later kernels with three config options. They are CONFIG_HZ_PERIODIC (the original method - all ticks all the time), CONFIG_NO_HZ_IDLE (default - no ticks only when a CPU is idle), and CONFIG_NO_HZ_FULL (the most recent development - no ticks all the time). At least one (boot) CPU will always get ticks. Ticks come back when there are multiple processes contending for the CPU.
  Applicability:   rhel5 vs rhel6

15.   An application team has moved a binary from an older hardware onto a newer system with a later model CPU. I notice the mention of AVX support on the hardware data sheet. What might this mean to me and what should I recommend to the application developer? --> what is AVX?
  Answer:   Advanced Vector Extensions (AVX) SIMD instructions for x86 instruction set architecture is supported by newer compiler (gcc 4.6 and above). Suggest the application owner to recompile. A fallback question might be to ask about SIMD - this should be recognized by most applicants.
  Notes:   This showed up in Sandy Bridge / Ivy Bridge as well as Bulldozer / Piledriver... all in early 2011 and later. Intel compiler picket is up at 11.1.
  Applicability:   Little understanding of compiler flags.

17.   I have this application that gets sequentially ordered UDP packets that is alerting about lost data. What should I check at the system level to see where this data loss is occurring?
  Answer:   nic stats (ethtool -S, sar -n EDEV with time stamp etc etc) to see if drops are at card level like crc, overrrun) or protocol level /proc/net/snmp (more complete) netstat -s or with time stamp like sar -n UDP
  Notes:   follow up question: what action can you take? increase ring buffer, replace the card, increasing socket etc etc depends on where the error is.
  Applicability:   Common network troubleshooting but the main aim to see if the errors are at protocol level if he thinks off SA mentality.
 
18.   There is this application on a system that keeps crashing. Error logs shows that the OOM killer is active. What is going on? --follow up--> If I wanted to find out who is likely to be killed, what would I look at?
  Answer:   OOM (out of memory killer) can happen when an application uses memory allocated in is virtual address space loading in the physical memory. If the memory is not big enough to allocate more the kernel try to kill processes as self protection from crashing. A good way is disable overcommitting. The "dstat --top-oom" can tell me who is a candidate for killing. /proc/PID/oom_score is your value as viewed by the OOM killer. The higher the score, the more likely you are to be killed. This value can be adjusted via /proc/PID/oom_score_adj using values between -1000 and 1000. (/proc/PID/oom_adj is the old interface.)
  Notes:  
  Applicability:   OOM kills based on resident memory, not virtual memory.
 
19.   I have zombies on my system. What system call should I me making to prevent this?
  Answer:   The wait() calls (wait() / waitpid()) reap zombies.
  Notes:   This might be too obvious of a question - ask and find out.
  Applicability:   I frequently ask about the wait() call directly. Most confuse it with yield - as in "wait for I/O" or a process is "in a wait state". This is just a different way of asking - and applicants should know where zombies come from and how to kill them.
 
20.   I just successfully malloc()d a gig of memory into my application. I see that my "size" has changed but my "RSS" has not changed. What is going on?
  Answer:   malloc() increases sz and vsz. Actually dirtying the pages effects rss.

Retired, Deprecated, & Trial Questions
xx.   [Trial] In IPv4 networking, what is an IP Address, a Subnet, and a Netmask? What is the relationship between them?
  Answer:   At a high level, the Subnet and Netmask define a scope that incorporates several IP Addresses. The relationship is one of the criteria used by IPv4 networks to help determine whether a source node and a destination node are members of a common local network, or if they are remote from one another. It further helps determine if packets should be delivered by Layer 2 (switched) or Layer 3 (routed) means. Digging deeper: the binary AND of the IP Address and the Netmask is a mathematical transformation that yields the Subnet. This relationship can be used to determine if any 2 addresses are members of the same Subnet.
  Notes:   The IP addresses in a subnet are always contiguous. There are specific mathematical rules governing the construction of a legal Netmask.
  Applicability:   Topic is fundamental to understanding how IPv4 networks work and how packets are routed.

xx.   An earlier question referred to static/dynamic linking... Which is preferred for a binary? What is the most popular / acceptable method today? Why?
  Answer:   Dynamic is by far the more popular method. Static is rarely used (except possibly for /sbin & /bin binaries (on some Unicies) - and this is now rare). The benefits of dynamic linking are generally seen with faster load times (object code is already in memory), and memory saving. The "negative" side effect is performance / locality of code - this is in most cases statistically insignificant.
  Notes:   The way this is worded references the assumption that static/dynamic linking was covered in the previous ldd question.
  Applicability:   This should be common knowledge for an admin who has ever supported non-COTS applications. The way it is asked is a bit unfair for AIX admins - as this is just so non-typical for and AIX admin to encounter. I think it is more indicative of an admin that has cross-Unix experience. (This is a question left over (unchanged) from the admin interview.)
